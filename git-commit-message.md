# Git Commit 消息规范

## 提交格式

```
<type>(<scope>): <subject>

<body>

<footer>
```

## 类型 (Type)

| 类型 | 描述 |
|------|------|
| feat | 新功能 |
| fix | 修复 bug |
| docs | 文档更新 |
| style | 代码格式调整（不影响代码运行） |
| refactor | 重构（既不是新增功能，也不是修改 bug 的代码变动） |
| perf | 性能优化 |
| test | 测试相关 |
| chore | 构建过程或辅助工具的变动 |
| revert | 回滚之前的提交 |

## 示例

### 新功能
```
feat(login): 添加用户登录功能

- 实现用户名密码登录
- 添加验证码验证
- 集成 JWT 认证
```

### 修复 bug
```
fix(auth): 修复登录超时问题

- 增加超时重试机制
- 优化网络请求超时设置
```

### 文档更新
```
docs(readme): 更新项目说明文档

- 添加快速开始指南
- 更新 API 文档
```

## 规范要求

1. 首字母小写
2. 使用现在时态
3. 用中文描述，不超过 50 个字符
4. 正文使用换行分隔，每行不超过 72 个字符
5. 说明"做了什么"和"为什么"，而不是"怎么做"
